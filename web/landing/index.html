Public invariant

I -> I_system^S -> H_lex -> G_lex -> I_lexicon

Synthia keeps lexicon indeterminacy distinct from generic uncertainty. The chain preserves which vocabulary is active, how strongly a document belongs to it, and what unresolved ambiguity must stay visible for human review.

What Synthia is now

A research core for classification memory

Synthia is not yet a large model, autonomous agent, field robot, or production platform. Its current value is architectural: it remembers which lexicon it is using, why a term belongs there, what evidence supports it, and where uncertainty or contradiction enters the classification.

I_lexicon engine

The public math layer

Synthia computes lexicon state from a candidate distribution over lexicons: `P(L_i | d, S)`, where `L_i` is a lexicon, `d` is the document, term, or observation, and `S` is the active system context. 

P(L_i | d, S) = softmax(score(L_i, d))
H_lex = - sum(P(L_i) log(P(L_i))) / log(|L|)
G_lex = 1 - (P_top - P_second)
I_lexicon = bounded(alpha H_lex + beta G_lex + gamma contradiction_load)

`H_lex` measures distribution across possible lexicons. `G_lex` measures how weak the winning lexicon is against the second-best candidate. `I_lexicon` is the final context-preserving state used for classification, filtration, switching, or quarantine.

Applied directions

Lexicon memory across domains

Biology and PhyloCode direction

Synthia's first applied domain is biology: taxonomy, redescription, species-as-systems thinking, and PhyloCode-aware classification support. The goal is to help organize biological knowledge as living memory while preserving formal authorship and human authority boundaries.

Physics and mathematical classification

Synthia is designed to preserve specialized vocabulary in physics and mathematical systems without collapsing field-specific meanings into generic prose. FfeD-style structures, plithogenic logic, and future mathematical lexicons can each become separate context modules.

Future LLM protection

A future role is lexicon sentinel work: tracking which domain vocabulary an LLM is using, detecting taxonomy drift, preserving source-linked meanings, and warning when specialized language has been flattened.

Future field discovery

Swarm and field work remains simulation-first. Any biodiversity, drone, or field deployment would require legal authorization, ecological review, local collaboration, safety controls, and expert validation.

Codebase guide

What is inside the repository

Core modules

The Python core includes lexicon nodes, bridges, source scanning, soul generation boundaries, NSS math routing, neutrosophic kernels, taxonomy memory, swarm simulation, RethinkDB support, and HippoRAG-style trace storage.

CLI surface

Commands cover `sources`, `soul`, `lexicon`, `nss`, `plithogenic`, `taxonomy`, `swarm`, `hipporag`, and `codex` workflows. The full command surface is kept in the repository tests and docs so the public site can stay readable.

Private/public boundary

Public code and safe docs live in this repository. Private source ledgers, private correspondence summaries, implementation notes, and internal reports belong beside it in `Synthia_organisation`.

Development

Run tests from the repository root with `python -m pytest`. No raw private correspondence bodies, authentication caches, restricted manuscript bodies, or credential material should be committed.

Public boundary

Research model, not a production authority

Synthia can organize source-linked evidence and prepare human review packets. It does not replace taxonomists, redescriptors, museum workers, field biologists, journal editors, the ICZN, PhyloCode governance, conservation authorities, medical review, or any formal scientific authority.
